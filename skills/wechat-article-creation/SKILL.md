# 公众号文章创作（Workflow Skill）

## 核心原则（先读）

- **强制顺序**：遇到新产品/新概念/新技术，必须先做「信息搜索/资料收集」，再做「选题讨论」，最后才写正文；严禁跳过搜索直接开写。
- **不直接开写**：默认先给 3–4 个选题方向 + 大纲，让用户选一个再进入初稿。
- **公众号格式限制**：不要用 Markdown 链接格式 `[文字](URL)`，直接写 URL；少用分割线、少用加粗、避免过度结构化。
- **风格目标**：口语化开场 + 自问自答 + 心路历程 + 具体细节（时间/地点/人物/数字）+ 视野对比；避免官腔与“AI味”。
- **透明化决策**：对标题/选题/结构等关键决策，给出 1–3 句“为什么这么选”的理由，方便用户纠偏（不要输出长篇推理）。
- **核心原则不可妥协**：不编造数据与案例；不确定就先查证或向用户要材料；重要决策（选题/立场/引用/敏感表达）必须让用户确认。

## 快速开始（你需要先问清楚）

向用户收集并确认这些信息（不全就先补齐）：

- 目标受众是谁（越具体越好）
- 这篇要解决读者哪个问题/带走什么结论（1 句话）
- 字数范围（如：1200/2000/3000）
- 是否带产品/工具/概念（如有，必须做资料收集/搜索）
- 用户可提供的真实素材（建议至少 3 条）：时间/地点/人物/对话/截图/数据/踩坑经历

## 任务路由（先判断再执行）

同一个“写作需求”，可能是新写、改稿、审校、转 X 或快速咨询。先做任务路由，能显著减少返工与误判：

- 先按 `references/task-routing.md` 判断任务类型（A/B/C/D/E）。
- 需要真实测试/数据/截图的题：先建协作文档并等待用户补齐材料，再开写正文。

## 文件与目录约定（默认）

在用户当前仓库/项目中按需创建并使用：

- `_briefs/`：需求 brief
- `_knowledge_base/`：资料收集与搜索笔记
- `_协作文档/`：测试/配图/协作清单（需要时）
- `草稿/`：文章草稿
- `文章归档/YYYY/MM/`：成稿归档
- `images/`：配图（如果需要落地文件）

命名规范与模板见 `references/templates.md`。

## 工作流（按顺序执行）

### Step 1：保存需求 Brief（必做）

- 把用户需求整理成 brief，保存到 `_briefs/[项目名]-需求brief.md`。
- brief 必含：受众、核心信息、字数、特殊要求、可用素材清单。

### Step 2：信息搜索/资料收集（⭐必须优先执行）

- 对新产品/新概念/新技术：收集 **官方文档/产品介绍/技术特性/用户评价/竞品对比**。
- 把结果保存到 `_knowledge_base/[主题]-[时间].md`，并保留来源 URL（直接写 URL，不用 Markdown 链接）。
- **离线/无法联网时的替代**：向用户索要链接、截图、原文片段、或让用户先搜再把材料粘贴给你；你负责结构化整理与提炼结论。
- 搜索整理完成后，用 5–10 行向用户确认你理解的要点，再进入选题讨论。

### Step 3：选题讨论（必做）

- 提供 3–4 个选题方向（格式见 `references/templates.md`），每个包含：标题、核心角度、工作量评估（⭐/⭐⭐/⭐⭐⭐）、大纲（3–7 个一级标题）、优劣势。
- 让用户选一个，或基于反馈再迭代一轮选题。

### Step 4：学习风格（写前必做）

- 阅读 `references/style-guide.md`（10 条必须做到 + 10 条必须避免）。
- 从 `assets/` 随机挑 2–3 篇样本快速过一遍，抓住口吻、节奏、转折方式（不要照抄句子）。

### Step 5：创作初稿

- 用散文式段落推进（少标题/少列表），但逻辑必须清楚。
- 必须有：背景铺垫 → 冲突/问题 → 思考过程（自问自答）→ 对比/证据 → 你的选择 → 给读者的启发。
- 有个人素材库时，优先用真实经历/观点/案例做支撑（方法见 `references/personal-materials.md`）；素材不足先向用户索要，避免硬编。
- 提到关键概念/产品/数据处标注 `[图片]` 占位。
- 草稿保存到 `草稿/[标题]-草稿.md`。

### Step 6：三遍审校（交付前必做）

- 用 `references/review-checklist.md` 的细版清单做三遍审校（内容事实 → 风格去 AI 味 → 节奏与排版）。

### Step 7：爆款结构优化（可选但推荐）

- 从 5 种结构里挑一个套用并微调：问题-原因-方案-升华 / 故事-冲突-转折-洞察 / 颠覆认知-新视角-重建 / 对比-反差-原因-启示 / 数据冲击-深挖-案例-升华。
- 做完读率自检：开头前 100 字能抓住人；中段每 300 字有新洞察或转折；结尾有升华与行动引导。

### Step 8：配图清单（推荐）

- 建议 5–8 张（最少 3 张），为重要概念/界面/数据准备图；在文中标注 `[图片]`。
- 另附一份配图清单（位置/内容/要求/状态），模板见 `references/templates.md`。

### Step 9：长文转 X（可选）

- 把公众号长文改成 200–500 字标准版，或 1000 字+长推版（模板见 `references/templates.md`）。
