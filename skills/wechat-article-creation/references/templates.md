# 内容创作模板

## 概览

本文件提供公众号内容创作中常用的模板，包括选题讨论、协作文档、长文转X等场景的标准化格式。

## 选题讨论模板

### 标准格式

当需要讨论选题时，提供3-4个选题方向，每个包含以下要素：

```
## 选题方案 [序号]

**标题**：<吸引人的标题>

**核心角度**：<一句话说明这个选题的核心观点/切入点>

**工作量评估**：⭐/⭐⭐/⭐⭐⭐（1-3星，3星最复杂）

**大纲**：
1. <一级标题>
2. <一级标题>
3. <一级标题>
...
（3-7个大标题）

**优劣势分析**：
- 优势：<为什么这个选题好>
- 劣势：<可能的挑战或不足>
```

### 给品牌方的大纲要求

- **字数要求**：<300字
- **风格**：口语化
- **结构**：只有一级标题，不要子标题

### 示例

```
## 选题方案1

**标题**：我花了一周测试了5款AI写作工具，发现了一个有趣的现象

**核心角度**：通过对比测试5款主流AI写作工具，发现它们在不同场景下的优劣，并给出选择建议

**工作量评估**：⭐⭐⭐

**大纲**：
1. 为什么要做这个测试？
2. 我测试了哪些工具？
3. 测试过程和方法
4. 每个工具的表现（重点）
5. 我的最终选择和理由

**优劣势分析**：
- 优势：实用性强，对比全面，有数据支撑
- 劣势：测试工作量较大，需要真实使用体验
```

## 协作文档模板

### 标准格式

当需要协调测试、配图等任务时，创建协作文档：

```
# [项目名] 协作清单

## 测试任务清单

- [ ] <任务1>
- [ ] <任务2>
- [ ] <任务3>

## 配图需求清单

| 序号 | 位置 | 内容 | 要求 | 状态 |
|------|------|------|------|------|
| 1 | 开头 | 产品logo | 高清 | 待制作 |
| 2 | 第3段 | 产品界面截图 | 标注关键功能 | 待制作 |
| 3 | 第5段 | 数据图表 | 数据准确 | 待制作 |

## 时间成本预估

| 任务 | 预计时间 | 备注 |
|------|----------|------|
| 信息搜索 | 2小时 | 包含竞品对比 |
| 测试体验 | 4小时 | 需要真实使用 |
| 初稿撰写 | 3小时 | 约3000字 |
| 审校优化 | 2小时 | 三遍审校 |
| **总计** | **11小时** | |
```

### 示例

```
# Notion AI 深度体验 协作清单

## 测试任务清单

- [ ] 注册并体验Notion AI
- [ ] 测试写作辅助功能（续写、扩写、改写）
- [ ] 测试翻译功能
- [ ] 测试总结功能
- [ ] 对比Copy.ai和Jasper

## 配图需求清单

| 序号 | 位置 | 内容 | 要求 | 状态 |
|------|------|------|------|------|
| 1 | 开头 | Notion AI logo | 高清，官方提供 | 已获取 |
| 2 | 第2段 | 界面截图 | 标注AI入口 | 待制作 |
| 3 | 第4段 | AI写作演示 | 展示续写效果 | 待录制 |
| 4 | 第5段 | 对比表格 | 3个工具对比 | 待制作 |
| 5 | 第7段 | 使用场景 | 实际工作场景 | 待拍摄 |

## 时间成本预估

| 任务 | 预计时间 | 备注 |
|------|----------|------|
| 信息搜索 | 2小时 | 查看官方文档和用户评价 |
| 测试体验 | 4小时 | 深度使用3天 |
| 初稿撰写 | 3小时 | 约3500字 |
| 审校优化 | 2小时 | 重点检查流畅度 |
| 配图制作 | 2小时 | 5张图 |
| **总计** | **13小时** | |
```

## 长文转X模板

### 标准结构（200-500字）

```
<开头30-50字：金句/数据/价值主张>

<展开100-300字：具体细节、个人经历>

<结尾20-30字：互动引导>
```

### 示例

```
用了100+个AI工具后，我发现效率工具的终极形态其实很简单。

最近一周，我把常用的Notion、Obsidian、Roam Research都卸载了，只保留了一个工具。为什么？因为我发现工具越少，效率越高。就像上次我和朋友聊的，"工具应该为你服务，而不是你为工具服务"。

你正在用几个效率工具？
```

### X长文版（1000字+）

```
<章节1标题>
<内容>

——

<章节2标题>
<内容>

——

<章节3标题>
<内容>

...（更多章节）

<结尾开放式问题>
```

#### 重要规则

- 使用 "——" 分隔章节
- 重要人名第一次 @账号（如：@马斯克）
- 结尾要有开放式问题，引导讨论

### 示例

```
AI工具的3个陷阱

最近一个月，我测试了50+个AI工具，花了大概100小时。结果发现，大部分工具都不值得用。

第一个陷阱是"功能陷阱"。很多工具号称功能强大，但核心功能其实很弱。比如某写作工具，支持100种写作场景，但每一种都做不到专业。 @OpenAI 的GPT虽然功能单一，但每一项都做到了极致。

——

第二个陷阱是"集成陷阱"。很多工具试图把所有功能都集成进来，结果什么都做不好。就像瑞士军刀，功能很多但没有一个好用。专业的事情交给专业的工具，不要指望一个工具解决所有问题。

——

第三个陷阱是"依赖陷阱"。AI工具应该是辅助，不是替代。过度依赖AI，会失去思考能力。我有一个朋友，现在连写邮件都要AI帮忙，结果自己的写作能力反而退化了。

那么，应该怎么选AI工具？我总结了一套方法论：先问自己"这个工具能解决什么具体问题"，再看"这个问题是否必须用AI解决"，最后"这个工具的解决方案是否真的比其他方式好"。

你被哪个AI工具坑过？
```

## 文件命名规范

### 需求Brief文件

格式：`_briefs/[项目名]-需求brief.md`

示例：
- `Notion AI-需求brief.md`
- `GitHub Copilot-需求brief.md`

### 知识库文件

格式：`_knowledge_base/[主题]-[时间].md`

时间格式：`YYYY-MM-DD`

示例：
- `_knowledge_base/Notion AI-2026-01-09.md`
- `_knowledge_base/行业人物-李开复-2026-01-08.md`

### 协作文档

格式：`_协作文档/[项目名]-协作清单.md`

示例：
- `_协作文档/Notion AI-协作清单.md`
- `_协作文档/DeepSeek-协作清单.md`

### 草稿文件

格式：`草稿/[标题]-草稿.md`

示例：
- `草稿/我花了一周测试了5款AI写作工具-草稿.md`
- `草稿/为什么我卸载了所有AI工具-草稿.md`

### 成品文章

格式：`文章归档/[年]/[月]/[标题].md`

示例：
- `文章归档/2026/01/我花了一周测试了5款AI写作工具.md`
- `文章归档/2026/01/为什么我卸载了所有AI工具.md`

## 使用建议

1. **选题讨论**：不要跳过，先讨论再创作
2. **协作文档**：测试任务复杂时创建，简单任务可省略
3. **长文转X**：X版本要精炼，抓住核心观点
4. **文件命名**：严格按照规范，方便管理和检索
5. **定期整理**：知识库要定期更新和清理
