# 个人素材库（把“人味”做出来）

没有个人素材库，文章很容易“正确但无感”。个人素材库的作用不是堆信息，而是提供真实的经历、观点和细节，让内容读起来像你。

## 核心规则（先说清楚）

- 只用真实素材：时间/地点/人物/数字/对话，不能编造或夸大。
- 不够就要：宁可让用户补材料，也不要硬写“看起来像真的”的细节。
- 素材要改写：可以引用事实，但不要直接复制粘贴原文，改成文章叙事。

## 推荐的最小结构（够用就行）

你不需要很复杂，一个文件夹就能跑起来：

- `personal_materials/index.md`：主题索引（写过什么、在哪、对应文件）
- `personal_materials/raw/`：原始记录（如导出的 CSV、聊天记录、随手笔记）
- `personal_materials/cards/`：提炼后的素材卡片（每张卡 1 个经历/观点/案例）

如果你不想建目录，也可以直接让用户把素材贴过来：3 条真实经历 + 3 条观点/吐槽 + 1 组数据/截图。

## 素材卡片模板（建议）

### 经历卡（Experience）

```
标题：
时间：
地点：
人物：
触发事件（发生了什么）：
我当时怎么想（犹豫/冲突/转折）：
我怎么做（动作）：
结果（数据/反馈/教训）：
可公开程度（公开/可模糊/不可写）：
相关链接/截图（如有）：
```

### 观点卡（Opinion）

```
一句话观点：
我为什么这么想（1-3 条理由）：
反例/反对意见（我如何回应）：
我愿意为这观点付出什么代价（立场强度）：
可用在什么主题：
```

## 检索方法（给 AI 一个“找得到”的入口）

- 关键词检索：主题名/产品名/地名/人名/情绪词
- 时间检索：按年份/月份搜“上周/去年/某次”
- “我常说的话”检索：把口头禅/常用句式整理成列表（少量即可）

## 素材放在哪（三个最值钱的位置）

1. **开头引入**：用真实场景把读者拉进来（时间/地点/人物 + 一个冲突）
2. **观点支撑**：把抽象观点落到一两个真实案例（最好有数据/对比）
3. **结尾升华**：用个人洞察收束，留一个读者能接的开放问题

## 没有素材库时怎么写（降风险策略）

- 用“可替换占位符”：`【时间】` `【地点】` `【一个真实对话】`，提示用户补齐
- 只写你能确定的事实与判断，避免出现具体数字、具体案例
- 先出选题和大纲，等用户补素材后再写正文
