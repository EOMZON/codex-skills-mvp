# 公众号写作风格指南（口语叙事·去AI味）

## 概览

本指南定义公众号文章的风格要求，包括必须做到和必须避免的事项。严格遵循本规范，确保内容符合目标口吻。

## 必须做到（10条）

1. **自我定位明确**
   - 关键观点前强调身份："作为AI Native Coder..."
   - 明确表达自己的立场和角色

2. **自问自答式叙事**
   - 展示思考过程："为什么...呢？因为..."
   - 模拟真实的内心对话

3. **心路历程式叙事**
   - 展示决策过程："犹豫→思考→决定"
   - 让读者看到你的思考路径

4. **具体细节增强真实感**
   - 时间：具体到"上周三下午"
   - 地点：具体到"在星巴克"
   - 人数：具体到"和3个朋友"
   - 避免泛泛而谈

5. **背景铺垫充分**
   - 给读者足够的上下文
   - 不要直接进入正题，先说明背景

6. **对比丰富展示视野**
   - 提到多个竞品或选项
   - 展示你对行业的了解和思考

7. **标注清晰**
   - 每个案例都有链接（直接写URL）
   - 重要内容标注 [图片]
   - 引用具体来源

8. **口语化开场**
   - 用"说起来"、"其实"、"倒是"开头
   - 营造对话感，拉近距离

9. **建立内容连续性**
   - 引用之前的作品或观点
   - "之前我写过..."
   - "这是我在xxx文章中提到的..."

10. **数据后加个人判断**
    - 不要只列数据，要加评论
    - "这个数据还挺猛"
    - "这个数字让我有点意外"

## 必须避免（10条）

1. **过度结构化**
   - ❌ 不要用过多的小标题、列表
   - ✅ 用散文式段落，自然流动

2. **直接陈述无过程**
   - ❌ 不要直接说结论
   - ✅ 展示思考路径，如何得出结论

3. **细节不够具体**
   - ❌ 避免"最近"、"一些"、"很多人"
   - ✅ 用具体的时间和数字

4. **缺少图片标注**
   - ❌ 提到概念、产品时不标注 [图片]
   - ✅ 重要内容都要有配图标注

5. **标题过于正式**
   - ❌ 不要用《关于XXX的研究》这种学术标题
   - ✅ 用口语化、吸引人的标题

6. **技术细节过度**
   - ❌ 不要陷入技术细节的泥潭
   - ✅ 抓住核心，点到为止

7. **商业推广不真诚**
   - ❌ 不要硬广，明显推销
   - ✅ 真实体验，理性分析

8. **分割线过多**
   - ❌ 不要用过多的 `---` 分割线
   - ✅ 小标题本身就是分隔

9. **加粗斜体过多**
   - ❌ 不要到处用 `**` 和 `*`
   - ✅ 只在极少数关键处使用

10. **Markdown链接格式**
    - ❌ 不要用 `[文字](链接)` 格式
    - ✅ 公众号不支持，直接写URL

## 风格示例对比

### 开头方式

❌ 错误示例：

```
AI工具的发展越来越快，今天我们来介绍一款新的AI写作工具。
```

✅ 正确示例：

```
说起来，上周三在星巴克和几个朋友聊天，大家都在讨论AI工具。作为AI Native Coder，我对这个话题还挺有感触的。
```

### 表达观点

❌ 错误示例：

```
这款工具很好用，效率提高了30%。
```

✅ 正确示例：

```
这个数据还挺猛的。我原本以为效率提升也就10-20%，没想到能达到30%。为什么这么高呢？我想可能是因为...
```

### 提到竞品

❌ 错误示例：

```
市面上有一些类似的工具。
```

✅ 正确示例：

```
我对比了Notion AI、Copy.ai、Jasper这几个工具，各有各的特点。Notion胜在集成，Copy.ai价格亲民，但Jasper在长文写作上确实更强。
```

## 应用原则

1. **整体把控**：先看全局，是否符合目标口吻
2. **逐条检查**：对照10+10条要求，逐项检查
3. **读起来像不像**：想象你本人在说这话，感觉对不对
4. **真实感**：是否有具体的细节、时间、地点、人物
5. **思考过程**：是否展示了你的思考和决策过程

## 常见问题

**Q: 可以用emoji吗？**
A: 可以少量使用，不要过度。

**Q: 可以引用别人的观点吗？**
A: 可以，但要标注来源，最好有链接（直接写URL）。

**Q: 技术文章可以写得专业一点吗？**
A: 可以，但要避免陷入细节，抓住核心，用通俗语言解释。
